Phone Comparison

OVERVIEW

Phone Comparison is a website for comparing fundamental smartphone specifications for people who are looking to purchase a new phone and those who are not particularly well-versed in technical jargon regarding smartphones. The site uses PHP and an Apache web server for the back-end with a MySQL database while the front-end was coded with HTML, CSS and JavaScript.

SKILLS: Ideation, coding, debugging

TOOLS: Sublime Text Editor, XAMPP, phpMyAdmin

CONCEPT DEVELOPMENT

For this project, I had to design a website that would make use of a database containing content that users could view and filter through. I decided to create a site that would allow users to compare phones and their specifications when searching for a new device. Existing websites such as GSMArena that have a database of smartphones go into very specific details for each device and use jargon that can be confusing for those who aren't tech savvy.

I wanted to create a similar site, but simplify the information and break it down into some of the most basic specifications and features that people tend to look at when considering their smartphone options. By narrowing down significant details of each phone, it will be easier for users who are unfamiliar with technical jargon to learn about some basic specs people tend to look at (eg. random access memory) and also aid them in finding a device that fits their needs.

DATABASE CREATION

Information on registered members and 31 different phones is stored and retrieved from a MySQL database using SQL queries. The database consists of 5 different tables: comments, member login information, member preferences, basic phone information (eg. name, manufacturer), and phone specifications (eg. screen size, battery capacity).

PROGRAMMING

The coding of the website was split into 3 parts. First was the overall front-end design using HTML, CSS, and the Bootstrap framework. This was followed by the creation and implementation of the database and content filtering, and implemented last was the user registration, login and preference saving as well as dynamically updating content on specific pages using AJAX and JavaScript.

REFLECTION

The completed website has 6 main pages that users can access through the navigation: index, browse, specifications information, compare, registration/login and account. Users can browse the site's database of smartphones to stay updated on phones released in recent years and can also compare and contrast the basic specifications of each phone on the website with the compare feature.

This project taught me a lot about back-end aspects of web design and development. In particular I found that database design was quite complex and involves much consideration on the relations between tables and how data would be stored and accessed. For example, the basic phone information and phone specifications tables have to share a key column in order to write out a SQL statement that will access information about a specific phone from both tables. If the tables don't have proper relations with each other, it becomes difficult to write out an appropriate SQL query. Previously I also had very little knowledge about AJAX and dynamically updated content, so this project allowed me to broaden my skills and knowledge with regards to both front-end and back-end web development.
