SLEEP ON IT

OVERVIEW

Sleep on It is a mobile application for helping users keep track of their sleeping habits with three main features: a clock alarm, a countdown timer, and a stopwatch timer. The application keeps a log of hours slept; the amount of sleep a user gets affects the growth of a star used to visually represent the data.

TASKS: Concept planning, coding, debugging

CONCEPT DEVELOPMENT

This application was inspired by another application we found on the Google Play store that helps the user track their productivity and how long they have studied for. The longer they spend studying, the larger a tree within the app grows. We decided to adapt the growth concept to a sleep tracker and alarm related application where you can grow stars while you sleep. We also thought that a sleep tracker might be something that students like ourselves may find useful when trying to fix our abysmal sleep schedules or see just how bad they really are. â€‹ We decided to have three different types of alarms and timers to allow users to track how long they sleep for depending on their needs. The clockis used when the user has a specific time they need to wake up at, while the countdown timer can be used when the user wants to sleep for a set period of time (eg. power nap for 30 minutes). Lastly, the stopwatch timer is used when the user wants to sleep indefinitely, but keep track of how long they sleep for in total. Data regarding how long the user has slept for is saved within a database and viewed through a sleeping log screen.

PROGRAMMING

For this application, I was responsible for coding the bulk of the application's back-end features while my teammates focused more on the front-end and layout. I coded the alarm clock feature and the log of sleep sessions. For the back-end, I coded the storage of data from each alarm and timer into a MySQLite database, as well as accessing and displaying data from the database. I also compiled the other features coded by my teammates into a single application and added extraneous features such as notifications and sound.

The alarm clock allows the user to set a specific time that they want to sleep until. When the alarm (or other timers) goes off, a notification is displayed and sound is played. Tapping the notification opens up the app and allows the user to turn off alarm while simultaneously saving the data of the sleep session (eg. date, amount of time slept, growth size of the star) to the database.

The log of sleep sessions is a list of items that each have the date and starting time of when the user went to sleep. Tapping on an items opens up a new activity screen that displays all the information of the sleep session (eg. date, amount of time slept, when the session started and ended) as well as a visual of the star that grew while the user was sleeping.

REFLECTION

Some of the problems we ran into mainly involved the use of services to allow some of the application's features to continue running in the background even if the application is closed. After the clock, countdown timer, or stopwatch is started, the amount of time elapsed and the growth of the star in accordance to how much time passes should be able to continue being kept track of when the app is not open. Unfortunately, we were unable to make this work because the value of how much time passed was not being sent or received.

Regardless, we were successful overall in making a basic application that helps keep track of how long the user has sleeps for. We were able to implement the majority of the features we planned within the given time frame, including all of the basic functionality such as the alarms and timer as well as the saving and accessing of information to and from the database. If we had more time to work on the application, I would like to work on fixing up the appearance of the sleeping log to have it match what we initially visualized.
